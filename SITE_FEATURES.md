# ai-jobs-ko 사이트 기능 강화 후보

> 새 서비스가 아니라, 지금 https://jeonsworld.github.io/ai-jobs-ko/ 페이지에 *직접 얹는* 기능들.

전제
- **정적 사이트 (GitHub Pages)** 안에서 동작
- 새 백엔드·DB 도입 X. 추가는 *데이터 파일* 또는 *외부 공개 API* 까지
- 모든 기능은 한지·매거진 톤의 현 디자인과 정합

평가 축
- **공수**: S(반나절)·M(1~3일)·L(1주+)
- **임팩트**: ★~★★★
- **추가 데이터**: 빌드 시 한 번 만들면 되는 정적 파일

---

## 1. 즉시 시작 — Tier S (반나절)

### F1. 직업 딥링크 + 공유 가능한 URL
> `?job=회계사` 또는 `#cat=교육직&job=초등학교교사` 로 직접 진입. 트위터/링크드인 공유 시 정확히 그 카드를 띄운다.

- **사용자**: URL 복사·SNS 공유. 보던 직업이 새로고침 후에도 그대로.
- **구현**: 카테고리 진입·직업 선택 시 `history.pushState` 로 URL 동기화. 페이지 진입 시 `URLSearchParams`로 복원.
- **공수**: S
- **임팩트**: ★★ (PR·바이럴의 전제 조건)

### F2. 다중 직업 비교 모드
> 트리맵에서 ⌘+클릭으로 2~4개 직업을 *고정*. 우측 사이드 패널에 직업 카드들이 나란히. 임금·노출도·전망·학력 한눈에 비교.

- **사용자**: "백엔드 vs ML 엔지니어 vs 데이터분석가" 동시 비교
- **구현**: localStorage에 핀 상태 저장. 사이드에 비교 그리드 추가. 핀 직업은 트리맵에서 강조 외곽선.
- **공수**: S~M
- **임팩트**: ★★★ (직업 의사결정 핵심 UX)

### F3. 데이터 다운로드 / 임베드
> 페이지 푸터에 "전체 537개 직업 CSV / JSON 다운로드" + "차트 임베드 코드 복사".

- **사용자**: 기자·연구자·블로거가 데이터 그대로 사용 → ai-jobs-ko 자동 인용
- **구현**: data.json 그대로 다운, CSV는 Papa Parse로 즉석 변환, 임베드는 iframe snippet
- **공수**: S
- **임팩트**: ★★ (citation moat 형성, 거의 비용 없음)

### F4. 즐겨찾기 / 내 직업 리스트
> ★ 버튼으로 직업 즐겨찾기. localStorage 저장. 우측 사이드에 "내가 본 / 찜한" 섹션.

- **사용자**: 진로 고민 시 후보 모음
- **구현**: 사이드 패널 1개 추가, localStorage 키 1개
- **공수**: S
- **임팩트**: ★★ (재방문률 ↑)

## 2. 1~3일 — Tier M (큰 차별화)

### F5. 산점도 모드 — 노출도 × 임금 × 인구
> 레이어 토글에 "산점도" 추가. 트리맵 자리에 같은 영역에서 *x=AI 노출도, y=평균 연봉, 점 크기=취업자수, 색=카테고리*. 트리맵과 한 클릭으로 전환.

- **사용자**: "고임금 + 저노출" 사분면(찾고 싶은 안전 직업)을 즉시 발견
- **구현**: 캔버스에 같은 좌표계 사용. 사분면 라벨링. 한지 톤 유지.
- **공수**: M
- **임팩트**: ★★★ (이 데이터셋의 가장 강력한 인사이트 화면)

### F6. 의미적 검색 (Embedding-powered)
> 검색창에 "AI 만들기"·"사람 도와주기"·"손으로 만드는 일" 같은 *자연어*를 입력하면 의미적으로 가장 가까운 직업 5~10개. 단순 부분 문자열 매칭 너머.

- **사용자**: 직업명을 모르는 사람도 도달 가능
- **구현**: 537개 직업명+직무설명을 빌드 시 *사전 임베딩* (e.g., OpenAI embeddings 또는 multilingual-e5-small). 결과 JSON(직업당 384~768 dim)을 `embeddings.json`(<5MB)로 저장. 사용자 입력은 *클라이언트* 에서 transformers.js + 같은 모델로 임베딩, 클라이언트 cosine similarity.
- **공수**: M (`@xenova/transformers` 모델 로드 ~50MB이면 너무 큼; mini 모델 + 사전 양자화 권장)
- **임팩트**: ★★★ (Tier S 검색의 다음 진화. PR 헤드라인 가능)
- **대체 옵션**: 입력 임베딩만 외부 API(원티드 또는 OpenAI)로 부르고, 직업 임베딩은 정적 → 모델 로드 부담 0

### F7. 직업 비교 OG 이미지 (자동 공유 카드)
> 사이트가 `https://og.ai-jobs-ko.app/?jobs=회계사,간호사` 같은 URL 호출 시 PNG 자동 생성. 트윗·블로그에 임베드 시 멋진 카드.

- **사용자**: 공유한 게시물이 미려한 인포그래픽으로 표시 → CTR ↑
- **구현**: Vercel OG / Cloudflare Worker (정적이 아니라 작은 edge function 1개) 또는 빌드 시 *모든 직업 OG 미리 생성*하여 docs/og/회계사.png 형태 저장 (537장 = ~5MB)
- **공수**: M
- **임팩트**: ★★ (SNS 노출 시 효과)

### F8. URL 파라미터로 상태 = 페이지 = "데이터 스토리"
> F1 확장. 카테고리·레이어·핀 직업·검색어 모두 URL에 → "내가 만든 뷰"를 그대로 공유. 미디어가 기사에서 *링크 7개*로 데이터 스토리 작성 가능.

- **사용자**: 기자가 "지금 보세요 → 클릭"으로 독자 유도
- **구현**: F1과 함께 작업. 모든 상태 직렬화.
- **공수**: S~M
- **임팩트**: ★★ (콘텐츠 도구로의 변신)

## 3. 1주~ — Tier L (모트 형성)

### F9. 시계열 모드 — Career Time Machine (mini)
> 헤더 우측에 연도 슬라이더 (2015 → 2025). 슬라이드 시 트리맵이 부드럽게 morphing. 어떤 직업이 자라고 어떤 직업이 줄어드는지 한눈에.

- **사용자**: 25초만에 한국 노동시장 10년사를 본다
- **구현**: KOSIS 직업별 취업자 5~10년치를 빌드 시 추출 → `data_timeseries.json` (작은 시계열). D3 transition으로 squarify cell stable morphing (Treemap morphing은 까다로움 — `d3-hierarchy` 의 `treemap.tile(d3.treemapResquarify)` 활용).
- **공수**: L (1~2주, 핵심 도전은 cell stability)
- **임팩트**: ★★★ (Gapminder 같은 권위 있는 비주얼)

### F10. 이력서 한 줄 입력 → 내 위치 표시
> 헤더에 입력창 한 줄. "5년차 백엔드 개발자, Python·AWS" 입력 → 트리맵 위에 *나의 위치*가 동그라미로 표시 + "비슷한 사람은 X 직업으로 갑니다" (의미적 매칭).

- **사용자**: 거시 데이터를 "내 일"로 즉시 변환
- **구현**: F6의 임베딩 인프라 재사용. 입력 → 임베딩 → 가장 가까운 직업 1개 highlight + top-5 사이드 표시. *원티드 공개 API* 가 있으면 "원티드 풀에서 비슷한 사람 N명" 추가.
- **공수**: M (F6 후속이면 1~2일)
- **임팩트**: ★★★ (개인화 = 재방문 + 공유)

### F11. 인접 직업 / 직업 그래프 미니뷰
> 직업 카드 디테일에 "이 직업의 이웃" 작은 force-directed 그래프 (5~10 노드). 임베딩 거리 기반 5개 인접 직업 + 임금 격차·노출도 격차 화살표.

- **사용자**: 진로 분기점에서 즉시 전환 후보 파악
- **구현**: F6 임베딩으로 직업×직업 거리 사전 계산 → `neighbors.json`. 사이드에 50px 미니 SVG.
- **공수**: M
- **임팩트**: ★★ (디테일 UX 깊이)

### F12. AI 노출도 *내 평가* — 크라우드소싱 보정
> "당신이 평가하기엔 회계사의 AI 노출도는?" 0~10 슬라이더. 응답을 익명 집계해 사이트 평균과 휴리스틱 점수를 비교 표시.

- **사용자**: 자기 직업 노출도를 손으로 만져보면서 사이트와 *대화*
- **구현**: 응답 저장은 Cloudflare KV / Supabase 무료 tier 또는 GitHub Discussions API (정말 정적 유지하고 싶다면). 그래프에 "사용자 평균" 라인 추가.
- **공수**: L (가벼운 백엔드 1개 추가하지만 SaaS 무료)
- **임팩트**: ★★★ (참여형 데이터셋 자체가 자산. 분기 리포트의 메인 콘텐츠)

## 구현 순서 가이드

새 데이터·인프라 의존성 그래프

```
F1 딥링크 ─┬─ F8 상태 공유
           └─ (모든 후속의 전제)

F6 의미적 검색 (embeddings.json 1개 생성) ─┬─ F10 이력서 매칭
                                          └─ F11 인접 그래프

F9 시계열 (data_timeseries.json) ─ 독립

F12 크라우드 노출도 (외부 KV 1개) ─ 독립
```

빌드 파이프라인 추가 단계
1. `python3 build_data.py` — 현재 (변경 없음)
2. `python3 build_embeddings.py` — *신규* (537 직업명+설명 → 임베딩 .json, 5MB 미만)
3. `python3 build_timeseries.py` — *신규, 선택* (KOSIS 5~10년 추출)
4. 빌드 산출물을 `docs/`에 복사 → `git push`
